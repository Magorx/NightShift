### Bug Fix: Item drop spawns PhysicsItems instead of GroundItem
- **Problem**: Dropping items from inventory (UI drag-drop, death drop, building popup overflow) created a single non-physical `GroundItem` for the entire stack. Items didn't behave as real physics objects and stacks looked like one item.
- **Root cause**: `_spawn_ground_item()` in `player.gd` instantiated a `GroundItem` scene (static Node3D) instead of individual `PhysicsItem`s (RigidBody3D).
- **Fix**: Rewrote `_spawn_ground_item()` to spawn one `PhysicsItem` per unit with small random XZ offsets. All three callers (UI drag-drop, death drop, building popup overflow) now produce real physics objects that scatter.
- **Test update**: `sim_player.gd` test 9 updated to check `physics_items` group instead of `ground_items`, removed `_hovered` dependency.
