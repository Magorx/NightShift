# Destroy Mode 3D Shader Overhaul

**Date**: 2026-04-05 evening (22:49 – 23:30)
**Duration**: 0.7h

## Work Done

Rebuilt the destroy mode visual system for 3D. The old 2D canvas_item shader was non-functional after the 3D transition.

### New shaders
- `destroy_highlight_3d.gdshader` — spatial overlay with animated diagonal red lines (screen-space, slow-moving)
- `destroy_outline_3d.gdshader` — inverted hull silhouette outline (cull_front + vertex normal expansion)
- `destroy_cursor_3d.gdshader` — flat ground cell with red fill and dark-red border

### build_system.gd changes
- `_get_visual_nodes()` now finds MeshInstance3D in Model subtree (was Sprite2D/ColorRect)
- Highlights use `material_overlay` + `next_pass` instead of replacing base materials
- Cursor cell uses ArrayMesh built with SurfaceTool — one quad per cell at correct terrain height
- Drag area highlights all cells in the rectangle, not just cursor position
- Removed all stale 2D code (frame UV tracking, AnimatedSprite2D refs, _draw stub)
- Select highlight (info panel) also ported to 3D material_overlay
